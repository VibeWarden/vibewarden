MIME Sniffing  Low Mitigated by VibeWarden

Browsers that ignore the declared Content-Type and infer the type from the content body can be tricked into executing a malicious payload as a script. X-Content-Type-Options: nosniff tells the browser to trust the declared type.

What is MIME Type Sniffing?

Early browsers tried to be helpful: if a server sent a file with a wrong or missing Content-Type, the browser would "sniff" the content and guess the real type. A text file containing <script> tags might be executed as HTML. A file served as text/plain but beginning with a valid MIME signature might be executed as an image or application.

This is a problem when users can upload files. A user uploads a file named profile.jpg that actually contains JavaScript. If the server serves it as text/plain without nosniff, older browsers execute it as a script in the origin's context.

Modern browsers respect X-Content-Type-Options: nosniff and refuse to sniff. Without this header, the attack surface remains even in 2025 for some browser/extension combinations.

The Vulnerable Endpoint

GET /vuln/mime-sniff serves JavaScript content but declares Content-Type: text/plain:

// INTENTIONALLY VULNERABLE — wrong Content-Type for JS content
func handleMIMESniff(w http.ResponseWriter, r *http.Request) {
  w.Header().Set("Content-Type", "text/plain")
  fmt.Fprint(w, "alert('MIME sniffing attack')")
}

Without nosniff, a browser that sniffs the content might recognise it as JavaScript and execute it when loaded via a <script src="..."> tag.

VibeWarden Protection

X-Content-Type-Options: nosniff

VibeWarden adds this header to every response from your app:

X-Content-Type-Options: nosniff

This single header instructs all modern browsers to:

  • Never execute a response as a script unless its Content-Type is a valid JavaScript MIME type (text/javascript, application/javascript, etc.).
  • Never interpret a response as a stylesheet unless its Content-Type is text/css.
  • Trust the declared MIME type and never sniff the content.

Zero configuration required. VibeWarden applies this header automatically to every response regardless of what the application sets.
