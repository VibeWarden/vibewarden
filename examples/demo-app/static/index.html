What protects this app

VibeWarden is a security sidecar — a single binary that sits in front of your app and handles every layer of protection. The app behind it has zero security code.

TLS — Let's Encrypt

HTTPS is automatic. VibeWarden obtains and renews certificates via ACME. No configuration needed.

Auth — Ory Kratos

Session management, login flows, and registration are handled by Ory Kratos. The app only sees injected X-User-* headers.

Social Login

When social providers (Google, GitHub, GitLab, Discord) are configured in vibewarden.yaml, sign-in buttons appear automatically on the login and registration pages — no code changes required.

Rate Limiting — 20 req/s per IP

Requests over the threshold receive 429 Too Many Requests with a Retry-After header. Try the Rate Limit Test page.

Security Headers

Every response carries: HSTS, CSP, X-Frame-Options, X-Content-Type-Options, and Referrer-Policy. Inspect them on the Headers Inspector.

AI-Structured Logs

Every event is logged with schema_version, event_type, ai_summary, and payload. Machine-readable by design.
