Vulnerability Lab

This app has intentional security vulnerabilities. VibeWarden mitigates most of them. Try to exploit them.

Each page below demonstrates a real vulnerability class. Where VibeWarden provides mitigation, the protection is applied automatically — no code changes needed in the app. Where it does not, the demo shows what can still go wrong and what you should fix at the application layer.

Reflected XSS

High

User-supplied input is reflected directly into the page without escaping, allowing arbitrary script execution in the victim's browser.

✓ Mitigated by VibeWarden (CSP blocks inline scripts from untrusted sources)

Stored XSS

High

Malicious scripts are persisted in the database and executed whenever the affected page is loaded by any user.

✓ Partially mitigated by VibeWarden (CSP reduces impact; app must sanitise input)

SQL Injection

High

Unsanitised query parameters are interpolated directly into SQL statements, allowing data exfiltration and database manipulation.

✗ Not mitigated by VibeWarden (fix at the application layer: use parameterised queries)

Clickjacking

Medium

The page can be embedded in a hidden iframe, tricking users into clicking UI elements they cannot see.

✓ Mitigated by VibeWarden (X-Frame-Options: DENY and CSP frame-ancestors)

CSRF

Medium

A forged cross-origin request can trigger state-changing actions on behalf of an authenticated user without their knowledge.

✓ Mitigated by VibeWarden (Kratos session cookies carry SameSite=Strict)

Open Redirect

Medium

An unvalidated redirect parameter allows attackers to redirect users to malicious sites using a trusted domain as a stepping stone.

✗ Not mitigated by VibeWarden (validate redirect targets at the application layer)

Information Disclosure

Low

Verbose error messages, stack traces, or misconfigured headers expose internal details that aid further attacks.

✓ Partially mitigated by VibeWarden (security headers strip server banners; app must suppress stack traces)

MIME Sniffing

Low

Browsers that ignore declared Content-Type and infer the type from content can be tricked into executing a malicious payload as a script.

✓ Mitigated by VibeWarden (X-Content-Type-Options: nosniff on every response)
