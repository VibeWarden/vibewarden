Information Disclosure  Low Mitigated by VibeWarden

Debug endpoints left exposed without authentication reveal Go internals, environment variables, and configuration secrets. VibeWarden's auth-on-all-paths default blocks unauthenticated access automatically.

What is Information Disclosure?

Information disclosure occurs when an application reveals data that should be kept private — stack traces, debug variables, database connection strings, API keys, internal service URLs. Even "low-severity" disclosures can give an attacker everything they need to escalate an attack.

A common scenario: a developer adds a /debug/vars endpoint to troubleshoot a production issue, commits it without thinking, ships it, and forgets to remove it. The endpoint is never intentionally exposed, but it is reachable because there is no auth check.

The Vulnerable Endpoints

This demo app has two intentionally vulnerable debug endpoints. The developer forgot to secure them, but VibeWarden catches the mistake:

GET /vuln/debug/vars Go version, goroutines, memory stats, env vars, uptime
GET /vuln/debug/config Fake DB URL with password, fake API keys, internal service URLs

Neither endpoint is listed in public_paths in vibewarden.yaml, so VibeWarden requires authentication for both.

What Would Be Exposed Without VibeWarden

If VibeWarden were not protecting these endpoints, an unauthenticated attacker would receive responses like these:

{ "go_version": "go1.23.4", "goroutines": 7, "uptime": "2h34m12s", "alloc_bytes": 1048576, "env_vars": [ "PORT=3000", "DATABASE_URL=postgres://admin:realpassword@prod-db:5432/app", "STRIPE_KEY=sk_live_REAL_KEY", ... ] }
{ "database_url": "postgres://admin:password123@internal-db:5432/production", "stripe_secret_key": "sk_live_FAKE_KEY_FOR_DEMO_PURPOSES_ONLY", "internal_api_url": "http://internal-payments-service:8080", "smtp_password": "smtp_FAKE_PASSWORD_DEMO" }

VibeWarden Protection

Auth on all paths by default

VibeWarden requires authentication for every path that is not explicitly listed in public_paths. The debug endpoints are not listed, so every unauthenticated request is rejected before it reaches the app handler.

# vibewarden.yaml — only explicitly public paths bypass auth public_paths: - /health - /public - /static/* - /vuln/* # vuln lab paths are public for demo purposes

VibeWarden also sets X-Content-Type-Options: nosniff on every response, preventing browsers from misinterpreting JSON as a different MIME type that could be exploited via MIME sniffing.

Defence in depth: even if auth is bypassed (e.g. by an insider), sensitive data should never appear in debug endpoints. VibeWarden provides a safety net for the mistake; the correct fix is to remove debug endpoints before shipping or gate them behind a separate secure channel.
