⚠️ The endpoint GET /vuln/redirect?url=... is INTENTIONALLY VULNERABLE. It redirects to any URL without validation. Do not deploy this in production.

Open Redirect  Medium Not Mitigated by VibeWarden

An unvalidated redirect lets attackers craft a URL on your trusted domain that silently redirects users to a phishing site. VibeWarden cannot prevent this — it is an application-level vulnerability.

What is an Open Redirect?

Many apps redirect users after login, logout, or completing a flow: /?next=/dashboard. If the next (or url, redirect) parameter is not validated against an allowlist, an attacker can set it to an external URL. The user follows a link that starts on your trusted domain and ends up on a phishing page.

Open redirects are commonly used in phishing campaigns because the initial URL looks legitimate. For example:

https://trusted-bank.example.com/login?next=https://evil-phishing.example.com/login

After clicking the link, the user is redirected to the attacker's fake login page that looks identical to the real one.

The Vulnerable Endpoint

// INTENTIONALLY VULNERABLE — redirects to any URL without validation
func handleOpenRedirect(w http.ResponseWriter, r *http.Request) {
  target := r.URL.Query().Get("url")
  http.Redirect(w, r, target, http.StatusFound)
}

VibeWarden's Honest Limitation

VibeWarden cannot validate redirect targets

VibeWarden is a reverse proxy, not a WAF. It does not inspect application logic such as redirect parameters. An open redirect vulnerability lives entirely inside the application code, where only the application knows which destinations are legitimate.

This is deliberate. A sidecar cannot safely apply rules about your app's business logic without risking false positives that break legitimate flows.

The correct fix is in your code. VibeWarden cannot do it for you.

How to Fix This in Your App

Validate redirect targets against an allowlist

The safest approach is to reject any redirect that leaves your domain:

// VULNERABLE — accepts any URL
target := r.URL.Query().Get("url")
http.Redirect(w, r, target, http.StatusFound)
// SAFE — only allow relative paths or known hosts
func safeRedirect(w http.ResponseWriter, r *http.Request, fallback string) {
  target := r.URL.Query().Get("url")
  parsed, err := url.Parse(target)
  if err != nil || parsed.Host != "" {
    // External URL — refuse and fall back to a safe default.
    http.Redirect(w, r, fallback, http.StatusFound)
    return
  }
  http.Redirect(w, r, parsed.Path, http.StatusFound)
}

Alternatively, use an opaque token (e.g. a session key mapping to a pre-approved URL) so the redirect destination never appears in the URL at all.
